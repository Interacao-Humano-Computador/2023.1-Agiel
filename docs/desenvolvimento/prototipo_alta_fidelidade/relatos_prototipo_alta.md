# ***Relato de Resultados da Avaliação do Protótipo de Alta Fidelidade***

## **1. Introdução**
<p align="justify">
&emsp;&emsp;Este documento apresenta o <b>Relato dos Resultados</b> da <a href='https://interacao-humano-computador.github.io/2023.1-Agiel/desenvolvimento/prototipo_alta_fidelidade/planejamento_alta_fidelidade/'>Avaliação do Protótipo de Alta Fidelidade</a> previamente planejada. O objetivo é <b>destacar as descobertas</b>, <b>fornecer dados</b> e <b>identificar áreas de melhoria</b>.
</p>
<p align="justify">
&emsp;&emsp;O relato dos resultados é essencial, pois documenta a perspectiva dos usuários. Compartilhar essas informações embasa decisões futuras e garante um produto final satisfatório.
</p>
<p align="justify">
&emsp;&emsp;Apresentaremos os principais insights, análises dos dados coletados e áreas de atenção para o próximo estágio de desenvolvimento. Nosso objetivo é promover melhorias tangíveis, impulsionando o <b>sucesso do produto</b> e a <b>satisfação dos usuários</b>.
</p>

## **2. Elementos do Relato**
### **2.1 Objetivos**
<p align="justify">
&emsp;&emsp;A avaliação do protótipo de alta fidelidade tem como principal objetivo <b>identificar problemas</b> na interação e interface do protótipo de alta fidelidade produzido pelo grupo. Algumas questões devem ser respondidas com a avaliação:
</p>
<ul>
<li> O usuário consegue realizar as tarefas no sistema?</li>
<li> O usuário consegue atingir o objetivo das tarefas? Com qual eficiência? Em quanto tempo? Após cometer quantos erros?</li>
<li> Qual parte da interface/interação o deixa insatisfeito?</li>
<li> O usuário entende o que deve fazer nas etapas das tarefas?</li>
<li> Quais problemas de IHC dificultam ou impedem o usuário de alcançar seus objetivos? Onde eles se manifestam? Com qual frequência ocorrem? Qual a gravidade deles?</li>
<li> Quais barreiras o usuário encontra para atingir seus objetivos?</li>
</ul>

### **2.2 Metodologia**
<p align="justify">
&emsp;&emsp;As avaliações foram realizadas por meio de entrevistas agendadas previamente de acordo com o <a href='https://interacao-humano-computador.github.io/2023.1-Agiel/desenvolvimento/prototipo_alta_fidelidade/planejamento_alta_fidelidade/'>planejamento</a> (planejamento_alta_fidelidade.md) e realizadas com devido consentimento do entrevistado. 
</p>
<p align="justify">
&emsp;&emsp;Nas entrevistas, os <b>protótipos produzidos</b> foram apresentados e, após isso, o entrevistador apresentou um <b>roteiro</b> do que deve ser realizado em cada tarefa para o entrevistado, o entrevistado por sua vez deve seguir o roteiro enquanto expressa seu pensamento em voz alta, ao final da tarefa o mesmo deve realizar um <b>feedback geral</b> sobre a tarefa realizada.
</p>
<p align="justify">
&emsp;&emsp;O escopo de participantes ultrapassou o estabelecido no <a href='https://interacao-humano-computador.github.io/2023.1-Agiel/desenvolvimento/prototipo_alta_fidelidade/planejamento_alta_fidelidade/#341-recrutamento-dos-participantes'>Planejamento da Avaliação</a>, com <b>cinco participantes</b> para o teste de usabilidade, e, consequente entrevista.
</p>

### **2.5 Problemas Encontrados**
<p align="justify">
&emsp;&emsp;Nesta seção esperamos que o avaliador após a realização da avaliação apresente os seguintes pontos:
</p>

#### **Tarefa: Obter dicas sobre Gestão de Carreira**
<p align="justify">
&emsp;&emsp;<b>Descrição dos problemas</b>:
</p>
<ul>
<li>(Problema 1) Usuários gostariam que o blog do estagiário tivesse mais destaque na página inicial.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Detalhes dos problemas</b>:
</p>
<ul>
<lo>(Problema 1) O problema ocorre quando o usuário tem que explorar a interface em busca de dicas para sua vida profissional.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Consequências/impactos do problema para o usuário</b>:
</p>
<ul>
<li>(Problema 1) O usuário pode acreditar que o site Agiel apenas oferece vagas de estágio e processos seletivos.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Gravidade dos problemas</b>:
</p>
<ul>
<li>(Problema 1) Apresenta gravidade leve pois o usuário ao realizar outras tarefas do site acaba descobrindo o blog.</li>
</ul>

#### **Tarefa: Cadastrar em Vaga**
<p align="justify">
&emsp;&emsp;<b>Descrição dos problemas</b>:
</p>
<ul>
<li> Os usuários se sentem confusos no momento em que são redirecionados para a tela de perfil após buscarem por vagas;</li>
<li> Os usuários gostariam de poder checar os dados do próprio currículo antes de confirmarem a inscrição para a vaga.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Detalhes dos problemas</b>:
</p>
<ul>
<li> (Problema 1) O problema ocorre quando o usuário tenta ir atrás de novas vagas e se depara com sua tela de perfil ao invés de ir diretamente para uma tela contendo todas as vagas disponíveis;</li>
<li> (Problema 2) O problema ocorre no ato de inscrição para nova vaga, os usuários gostariam antes de confirma a candidatura visualizar o próprio currículo para evitar possíveis erros.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Consequências/impactos do problema para o usuário</b>:
</p>
<ul>
<li> (Problema 1) O usuário pode ficar frustado ao não encontrar as vagas de primeira;</li>
<li> (Problema 2) O usuário pode ficar frustado em ter que voltar para seu perfil para checar suas informações.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Gravidade dos problemas</b>:
</p>
<ul>
<li>(Problema 1) Apresenta gravidade média pois o usuário após visualizar as outras opções encontra rapidamente o campo de vagas;</li>
<li> (Problema 2) Apresenta gravidade alta pois o usuário tem que voltar no fluxo para checar as informações multiplas vezes.</li>
</ul>

#### **Tarefa: Cadastrar Instituição de Ensino**
<p align="justify">
&emsp;&emsp;<b>Descrição dos problemas</b>:
</p>
<ul>
<li> O usuário considerou a seta do dropdown pouco intuitiva e visualmente insatisfatória.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Detalhes dos problemas</b>:
</p>
<ul>
<li> (Problema 1) O problema está na seta que abre o dropdown no topo da página inicial (header), pois seu design visual se assemelha a um "v" e não é intuitivo para os usuários.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Consequências/impactos do problema para o usuário</b>:
</p>
<ul>
<li> (Problema 1) A seta do dropdown não é apenas pouco intuitiva, mas também não possui um design visual atraente, o que afeta negativamente a experiência estética do usuário.</li>
</ul>
<p align="justify">
&emsp;&emsp;<b>Gravidade dos problemas</b>:
</p>
<ul>
<li> (Problema 1) A gravidade desse problema não é alta, pois se trata de um aspecto visual e de usabilidade específico relacionado à seta do dropdown.</li>
</ul>

### **2.6 Sugestões de Melhoria**

#### **Sugestões (Obter Dicas sobre Gestão de Carreira)**
<ul>
<li> (Problema 1) Inserir uma descrição ou um campo direto na home para acessar ou tomar conhecimento sobre o blog do estagiário.</li>
</ul>

#### **Sugestões (Cadastrar em Nova Vaga)**
<ul>
<li> (Problema 1) Ir direto para as vagas disponíveis no fluxo;</li>
<li> (Problema 2) Inserir uma preview do currículo do usuário antes da confirmação da participação;</li>
<li> (Problema 2) Permitir que o usuário edite o currículo na tela de confirmação de candidatura.</li>
</ul>

#### **Sugestões (Cadastrar Instituição de Ensino)**
<ul>
<li> (Problema 1) Substituir a seta do dropdown, tornando-a mais intuitiva e esteticamente agradável.</li>
</ul>
